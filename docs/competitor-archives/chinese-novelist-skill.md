- **状态**：craft.py `对话工艺` 已收（潜台词/停顿/权力博弈）。

### 2.6 悬念强度曲线 + 跨章三层弧（`hook-techniques.md` §强度等级/编排）— ✅ 已进 craft.py

- **强度 5 级**：好奇→关切→迫切→生存→终极，配读者反应 + 适用位置；**波浪式递进**「低-中-高-低-中-高-更高」，长篇只设 2-3 个最高强度锚点。
- **跨章三层弧**：短弧（2-3 章，即时满足）/ 中弧（5-8 章，情节波峰）/ 长弧（全书，核心谜题）。三弧必须同时运行，一弧结束时至少另一弧在高潮。多弧并行节奏示例（20 章）。
- **单章悬念三段式**：播种（前 20%）→生长（中 60-70%）→收获（后 10%）。
- **状态**：craft.py `SUSPENSE_CURVE` + `三层弧` 已收。

- **状态**：craft.py 偏「句段级工艺」（钩子/扩充/对话），**结构级模板（三幕/英雄之旅/单章四段）尚未系统进 craft.py**，是仍可借的一块。

### 2.8 悬念禁忌 + 章节间连接（`hook-techniques.md`）— 🔵 可补

5 禁忌：虚假悬念 / 机械降神 / 过度留白（每章至少回答一个旧悬念再提新的）/ 低风险钩子 / 同质悬念。伏笔与呼应（对称场景/重复对话/物品回归）；悬念升级递进；多线悬念（主线/人物/关系/时间四线并行）。

### 2.9 四阶段流程（SKILL.md + flows/）

三大黄金法则（展示而非讲述 / 冲突驱动 / 悬念承上启下）；phase0 中断续写检测 → phase1 三层递进问答（核心定位 Q1-3 / 深度定制 Q4-8 / 标题生成）→ phase2 规划+二次确认 → phase2.5 写作模式选择（串行/子Agent并行/Agent Teams）→ phase3「疯狂创作」（禁止向用户确认，一口气写完）→ phase4 自动校验修复（最多 3 轮）。

- **字数检查脚本**（`check_chapter_wordcount.py`）：剥 markdown 标记后只数 `[一-鿿]` 汉字，<3000 提示扩充。Windows 控制台 UTF-8 修复。

## 五、存档备忘（以后可能用，单列）

1. **情节结构模板**（plot-structures.md）是本项目尚未被 craft.py 吃干净的一块——三幕式章节分配表、英雄之旅 12 阶段、**单章内部四段结构（10/60/20/10）**、题材→结构→章节数选择指南，都是结构级工艺，shadow-verse 若要补「章节蓝图骨架」可直接吃（与 AI_NovelGenerator 六元数据卡片、ainovel-cli 弧型模板可三方对照）。
2. **扩充诊断表**（症状→技法映射）是把「扩写」从盲目变成「先诊断后下药」的好范式，可注入 narrate_revise。
3. **悬念禁忌 5 条**可作 checks.py 的「钩子质量反面规则」候选（虚假悬念/机械降神尤其值得做成检测）。
4. **分题材扩充富矿**（悬疑=线索分层 / 言情=情感微瞬间 / 奇幻=力量具身体验 / 历史=物质细节）是题材化写作工艺，与 webnovel-writer 题材 Profile 互补——一个管「数值阈值」，一个管「该写什么场景」。
5. 每式工艺都配**多题材实例**（悬疑/言情/奇幻/历史/都市/科幻），这批示例本身是高质量的中文写作语料，可作 craft.py 示例库扩充来源。
6. 本项目所有工艺文本是 GPL 友好的纯创作知识，吸收无依赖/许可负担（纯思路与文本工艺）。
